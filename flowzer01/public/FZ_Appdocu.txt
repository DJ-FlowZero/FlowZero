## Profile/Group Creation, Staging, and Publishing (2026-04)

### New Features & Learnings (2026-04-22)
- The Index Edit window now acts as a directory of all groups (profiles) and allows safe creation and publishing of new groups.
- The Add Entry and Download buttons have been removed to prevent accidental or incomplete additions.
- Only the [Create Group] and [Publish Group Files] buttons can add new groups, ensuring all associated files (profile, token, sticky) are created together.
- The profileId and fileName fields in the directory are now read-only to protect directory integrity; only description and extra fields are editable.
- A new [Save Changes] button allows you to persist edits to descriptions/comments in `fz_profile_index.json` by downloading/overwriting the file.
- All changes to the index are reflected in selectors and other UI components immediately after saving/publishing.
- UI improvements: clearer modal/box titles, increased spacing, and consistent button styling for a more user-friendly admin workbench.
- The workflow is now robust against accidental data loss or corruption and is ready for further user-facing UI development.

#### Example Group Creation & Publishing Workflow
1. Click [Create Group] in Index Edit. Three new files are created in `public/NewState/` with the next available number.
2. Edit the new group/profile as needed.
3. Click [Publish Group Files] to copy the staged files to `public/`.
4. Download and/or publish the updated `fz_profile_index.json` as needed.

### New Features & Learnings
- The text file editor (TextFileBackandSave.jsx) now supports editing and saving any `.txt` file by name.
- When saving, the filename is always taken from the React state (input field or file picker), so any valid `.txt` file can be created or updated in `public/NewState/`.
- When saving, if the file already exists, a backup is created with the `.bck` extension in the same folder.
- The refresh-to-public operation also supports any filename and creates a backup of the public file before overwriting (e.g., `public/text_2_test.txt.bck`).
- This means you can always roll back to the previous version by restoring the `.bck` file.
- The backend endpoints `/api/save-text` and `/api/refresh-text` now both accept a `filename` property in the POST body, defaulting to `text_text.txt` if not provided.
- This logic is robust for multi-file workflows and safe editing/publishing.

#### Example Workflow
1. Enter or pick a filename in the editor UI.
2. Edit and save: file is written to `public/NewState/` and backup is created if needed.
3. Refresh: file is copied to `public/` and backup is created if needed.
4. If a mistake is made, restore from the `.bck` file in either location.

# FlowZero Application Documentation (FZ_Appdocu)

## Overview
FlowZero is a zero-resistance workflow tool for managing and editing user configuration profiles (PUCKs), sticky notes, and personal tag tokens. The application is built with React and Vite, and operates on local JSON files for privacy and portability.

### Key Concepts
- **PUCK (Portable User Configuration Knowledge):** Local user profile JSON files, e.g., `fz0000.json`.
- **Sticky Notes:** Per-profile sticky note arrays, e.g., `fz0000_sticky.json`.
- **Tokens:** Per-profile tag tokens, e.g., `fz0000_token.json`.
- **User Index:** `fz_profile_index.json` and `fz_user_index.json` provide metadata and lookup for all profiles.

## File Structure
- `src/Jedit.jsx`: Profile (PUCK) JSON editor.
- `src/StickyEdit.jsx`: Sticky note editor for a selected profile.
- `src/TokenEdit.jsx`: Token/tag editor for a selected profile.
- `src/userIndexUtil.js`: Utility for loading user index.
- `public/`: Contains all user data files (JSON), config, and documentation.

## Main Components
### Jedit.jsx
- Loads user index and allows selection of a profile JSON file.
- Edits array of objects (fields: label, value, flag, todo) in the profile.
- Supports manual file open and save.

### StickyEdit.jsx
- Loads user index and allows selection of a profile's sticky note file.
- Edits array of sticky notes (fields: stickytype, note, links).
- Supports manual file open and save.

### TokenEdit.jsx
- Loads user index and allows selection of a profile's token file.
- Edits array of tokens (fields: Token, Meaning, Usage).
- No manual file open; always associated with a profile.

## Data Model
- Each profile file is a JSON object with a single key (profileId or similar) mapping to an array of entries.
- Sticky and token files follow the same pattern, with different field names.

## User Flow
1. User selects a profile from a dropdown (populated from user index).
2. The relevant file (profile, sticky, or token) is loaded and displayed in a form.
3. User can edit, add, or remove entries.
4. User can save changes as a new JSON file.

## File Naming Conventions
- Profile: `fzXXXX.json`
- Sticky: `fzXXXX_sticky.json`
- Token: `fzXXXX_token.json`
- User index: `fz_user_index.json`, `fz_profile_index.json`

## How to Rebuild
1. Install dependencies: `npm install`
2. Start dev server: `npm run dev`
3. Build for production: `npm run build`
4. Lint code: `npm run lint`

## Regeneration Guidance for AI
- Recreate the three main editors (Jedit, StickyEdit, TokenEdit) as React components.
- Each editor loads a user index, allows profile selection, and loads/edits the corresponding JSON file.
- Use controlled form inputs for editing arrays of objects.
- Provide file save (download) and, where appropriate, file open (upload) functionality.
- All user data is stored in the `public/` directory for local privacy.
- See this file and the codebase for field names and UI structure.

## Glossary
- **PUCK:** Portable User Configuration Knowledge (profile JSON)
- **Sticky:** Per-profile sticky notes
- **Token:** Per-profile tag tokens
- **User Index:** List of all profiles
