
## [2026-04-27] Directory Path Management

- All code now references the Gestalt directory via the FZ_GPATH variable in config.json.
- Changing the Gestalt directory only requires updating config.json and restarting servers.
- This pattern is recommended for all future directory or file path dependencies.

Data Model Overview
------------------
- fz_profile_index.json: Array of profile/group index entries (realName, profileId, fileName, description, type, tag, etc.)
- fzXXXX.json: { profile: [ { label, value, flag, todo, ... } ] }
- fzXXXX_token.json: { FZ_Tokens: [ { Token, Meaning, Usage, flag, ... } ] }
- fzXXXX_sticky.json: { sticky: [ { stickytype, note, links, flag, ... } ] }
- FZ_Flow_Config.json: { version, date, params: [ { id, label, description, options: [3], value } ], notes }
  - 18 params covering: response_length, default_mode, challenge_level, artifact_gate,
    explanation_depth, ambiguity_handling, scope_discipline, coding_strictness,
    confidence_display, session_close, initiative_level, tone_register,
    formatting_style, error_recovery, context_decay, limitation_acknowledgment,
    vocabulary_anchoring, context_priority

Key Utilities
-------------
- userIndexUtil.js:
  - getGestaltFileUrl(filename) → builds full fetch URL for any Gestalt file (SINGLE SOURCE OF TRUTH — use this)
  - fetchUserIndex()     → reads fz_profile_index.json, returns profile array
  - getFZ_GPATH()        → internal helper only; reads config.json FZ_GPATH. Do NOT call directly from components.

Key Workflows
-------------
1. Profile/Group Creation: Use IndexEdit to create a new group/profile, which generates new fzXXXX.json, *_token.json, *_sticky.json in NewState/.
2. Editing: Use Jedit, TokenEdit, StickyEdit, or TextFileBackandSave to modify data.
3. Calibrate Flow: Use the Calibrate Flow panel (Show Config button) to edit the 18 BioLogic behavioral params in FZ_Flow_Config.json.
4. Review Gestalt: Use Show Gestalt (modal) to preview a profile's full combined context at a given visibility level.
5. Publishing: Use IndexEdit to publish new/edited files from NewState/ to Gestalt/ and update fz_profile_index.json.
6. PUCK Export: Use Create PUCK to select rows from profile/token/sticky by visibility, optionally include the Flow Calibration block, and download as a .txt file ready for AI session priming. The export format is: loader readme + flow block (optional) + JSON data.
7. Backup/Restore: All saves create .bck files; Recyclebin/ holds old versions for manual recovery.

MAP Munchhausen Architecture Principle (Onboarding & Recovery)
-------------------------------------------------------------

Onboarding a New Developer
- Prerequisites: Node.js, npm, VS Code (recommended), Git
- Clone repo, run `npm install`, then `npm run dev` to start backend/frontend
- See FZ_Appdocu.txt and this file for architecture, code plan, and workflows

Recovery/Restore Instructions
- To restore any .json or .txt file, copy the corresponding .bck file (in NewState/, Gestalt/, or fz_txt/) to the original filename
- To recover deleted/overwritten profiles, copy from Recyclebin/ to Gestalt/
- For full reset, re-clone repo and copy in latest Gestalt/ and fz_txt/ files
