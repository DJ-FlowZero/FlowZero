# Sticky Editor

> **Stickies** are runtime modifiers — what is different *right now* about this entity.

Use stickies for anything situational, temporary, or currently active. They supplement or override the Profile for the current task without altering the stable base.

---

## Fields

| Field | Purpose |
|-------|---------|
| **Type** | Category — `note`, `gap`, `fault`, or any custom label |
| **Note** | The situational content |
| **Links** | Optional references — URLs, file paths |
| **Flag** | Visibility — `public`, `private`, or `secret` |

## Reserved types

| Type | Meaning |
|------|---------|
| `gap` | Information that is missing or unverified — flag for follow-up |
| `fault` | Active incident or known problem — visible in every PUCK until resolved |
