# Profile Editor

> The **Profile** holds the stable facts about a person, place, or entity — things that rarely change.

Each entry is a label/value pair with a visibility flag. Think of it as a structured background briefing — name, constraints, preferences, background.

---

## Fields

| Field | Purpose |
|-------|---------|
| **Label** | What the fact is — e.g. "Dietary constraint", "Location" |
| **Value** | The fact itself |
| **Flag** | Visibility — `public`, `private`, or `secret` |
| **Todo** | Optional follow-up note for incomplete or unverified entries |

## When to use

Fill in Profile first — it is the stable foundation. Stickies override it at runtime. The Flexicon names the things referenced inside it.
