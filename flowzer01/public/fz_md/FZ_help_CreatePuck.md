# Create PUCK

> **PUCK** = Portable Use Case Knowledge — a pruned, portable constraint packet compiled from your Gestalt.

A PUCK is what you hand to an AI, a colleague, or a tool at the start of a task. Instead of re-explaining yourself every session, you drop the PUCK and the receiver has exactly what they need — no more, no less.

---

## Workflow

**1. Select a Profile**
Choose which person's Gestalt to pull from. The dropdown lists all profiles in the index.

**2. Set Visibility**
Controls how much of the Gestalt is included:

| Level | What's included |
|-------|----------------|
| **Public** | Only public-flagged rows — safe to share openly |
| **Private** | Public + private rows — for trusted collaborators |
| **Secret** | All rows — full personal context |

**3. Select rows**
Each tab (Profile / Tokens / Stickies) shows the filtered rows. Tick only the rows relevant to the task — a PUCK should be *pruned*, not dumped.

**4. Include Flow Config** *(optional)*
Adds your Flow Calibration parameters — reasoning style, communication preferences, working modes.

**5. Export PUCK**
Downloads a `.txt` file — plain text, readable by any AI or human cold, no tools required.

---

## Tips

- Less is more. A PUCK with 5 precise rows outperforms one with 50 noisy rows.
- Stickies are the most powerful layer for short-term task context — use them for "what's different right now."
- Public visibility = safe to paste into any AI chat. Private/Secret = internal use only.
- PUCKs land in the PUCK Library once saved — reuse them in the Job Package Composer.
